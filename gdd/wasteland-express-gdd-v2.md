# 《废土快递》游戏策划案

> Wasteland Express — 2D 像素 Roguelike 经营 / 驾驶物流
> 引擎：HTML5 Canvas + Electron 桌面壳
> 文档版本：v2.0（2026-07 重写）
> 状态：3 派系 + 派系任务链 + 0 点碎片 + 4 人乘员 + 战斗 v3.0 已实现，存档系统跑通

---

## 一、为什么要做这个

玩了《FTL：Faster Than Light》和《暗黑地牢》之后，我一直想做一个"在废土里跑运输"的 roguelike。**但 FTL 是飞船、暗黑地牢是地下城，运输这件事在游戏里特别少**——它应该是废土世界观最自然的玩法之一。

**核心立意**：玩家不是英雄。玩家是**司机**。要算油钱、要应付劫匪、要交保护费、要选边站、要决定帮谁运货不帮谁。

目标用户：roguelike 玩家（FTL / 暗黑地牢 / 以撒）+ 经营玩家（缺氧 / 环世界）+ 废土世界观爱好者（《疯狂的麦克斯》）。

**为什么做像素风 + Electron 桌面**：2024 年像素风 roguelike 验证过（《雨中冒险 2》累计千万销量），但**带商业 + 经营深度的像素 roguelike 是空白**。Electron 壳子是为了 Steam 桌面发行方便，前端还是 Web。

## 二、核心循环

### 2.1 局内循环（一次跑货）

```
[选派系任务] → [选路线] → [启程] → [路上随机事件] → [到达城镇] → [交货/交钱/遭遇] → [回程或继续] → 回合+
```

每回合玩家做 3 个决定：
1. **接谁的活**（派系任务 vs 自由接单）
2. **走哪条路**（直路省钱有风险，绕路费油但安全）
3. **遇到事件怎么办**（战斗/交涉/贿赂/逃跑）

### 2.2 局外循环（跨回合）

```
[声望解锁] → [新任务线] → [新城镇] → [新载具部件] → [新乘员] → [新派系等级]
```

玩家的"局外资产"是：
- 派系声望（4 个派系，正负双向）
- 车辆改装（4 大类部件）
- 乘员羁绊（4 个固定乘员 + 招募的临时工）
- 0 点碎片（解锁收藏册 + 永久技能）

**重要决定**：没有"经验值/等级"。<u>不做 RPG 等级系统是项目早期决定</u>——理由是 roguelike 玩家讨厌"靠刷级变强"，要的是"靠选择变强"。

### 2.3 失败/通关

- **死亡/破产/被所有派系敌对** → Game Over
- **完成 4 派系主线**（4 个派系都达到 100 声望）→ Victory
- **隐藏结局**：收集所有 0 点碎片 + 不杀任何派系 NPC → 真结局

> 三个结局不是为了"延长游戏时间"，是为了给"3 种玩法"留空间：纯运输流（快胜利）、战斗流（多杀 NPC、慢胜利）、探索流（收集碎片、真结局）。

## 三、派系与声望系统

### 3.1 四个派系

| 派系 | 定位 | 好感行为 | 敌对行为 |
|---|---|---|---|
| **废土商队** | 中立 | 准时交货、保护商路 | 偷货、运违禁品 |
| **钢铁教会** | 秩序 | 缴税、剿匪、按时礼拜 | 拒绝礼拜、走私 |
| **自由民** | 自由 | 反抗压迫、帮弱小 | 欺负难民、帮权贵 |
| **零点破碎者** | 神秘 | 送 0 点碎片、神秘事件 | 透露秘密、破坏仪式 |

> **派系不是"任务列表"，是"立场系统"**。同一件任务，钢铁教会认为"对"的可能自由民认为"错"。<u>玩家在任务里做的每个选择都会双向影响两个派系</u>——这是核心驱动力。

### 3.2 任务链设计

每个派系 12-15 个任务，分 3 章：
- **第一章（声望 0-30）**：介绍派系，做简单任务，建立关系
- **第二章（声望 30-70）**：派系之间的冲突加剧，玩家必须选边
- **第三章（声望 70-100）**：派系秘密暴露，重大选择

> **踩过的坑**：早期派系任务都"线性"——完成任务自动加声望。后来改成"<u>同一个任务，根据玩家选择有 3 种声望变化路径</u>"。比如"运一批药到 X 镇"：给钢铁教会运 = 商队+5 / 教会+10；给自由民运 = 自由民+15 / 教会-5；拒运 = 商队-5 / 自由民+5。**这种"一个任务三种走向"让玩家感觉每个决定都有分量**。

## 四、车辆与改装

### 4.1 4 大类部件

- **车架（Chassis）**：决定载重上限、耐撞值。3 款（轻型/中型/重型）
- **引擎（Engine）**：决定最高速、爬坡、油耗。3 档
- **武器（Weapon）**：战斗中可用。机枪/霰弹/火箭筒/电磁脉冲
- **装甲（Armor）**：减伤。轻甲/中甲/重甲/反应装甲

每类 3-5 个等级，**总改装组合 4×3×4×4 = 192 种**（含部分不兼容）。

### 4.2 改装规则

- 改装需要**零件 + 工具 + 技能**。零件从废土城镇买或从 0 点碎片解锁
- **改装有副作用**：重装甲 = 慢 + 费油；重武器 = 后坐力让车难控制
- **没有"最强配置"**——每种玩法有最适合的改装方向

> 早期我们想做"车有 4 个槽位、每个槽位 8 种选 1，组合 8^4 = 4096 种"，后来砍到 192 种。<u>为什么</u>：4000+ 种组合玩家根本试不完，最终大家只玩"最优解"。192 种够有趣又不至于找不到方向。

### 4.3 油量与续航

- 油量 100%，每走一格 -1.5-3%（取决于车重 + 引擎档位 + 路面）
- 油耗是**核心资源管理**——油用光 = 抛锚 = 必死
- **油桶从城镇买**，价格随派系声望变化（商队友好 = 便宜，敌对 = 翻倍）

## 五、战斗与乘员

### 5.1 战斗系统

回合制 + 3 战术选择：
- **全力**（0）：高攻击，低防御
- **防守**（1）：低攻击，高防御
- **精准**（2）：中等攻击，必定命中要害

> 为什么不写回合制复杂伤害公式？因为玩家在乎"我打中了没"和"死了几个"，不在乎"伤害计算是否符合 RPG 公式"。**战斗 = 简单决策 + 资源管理（弹药 + 乘员血量）**。

### 5.2 4 个固定乘员（每个有独立性格 + 羁绊线）

| 乘员 | 角色 | 战斗定位 | 羁绊事件 |
|---|---|---|---|
| **凯西**（副驾） | 前商队会计 | 谈判（降低抢劫概率） | 教玩家算账（解锁技能"折扣"） |
| **雷**（机枪手） | 退伍士兵 | 火力输出 | 战后创伤事件（解锁"先发制人"） |
| **老张**（技师） | 退休机械师 | 改装效率 +20% | 教玩家隐藏改装位（解锁"秘密货舱"） |
| **小米**（跑腿） | 14 岁孤儿 | 侦察（提前看事件） | 寻找家人（支线任务） |

> **乘员不是技能点，是"人"**。每个有名字、有过去、会在路上说怪话。<u>这条线做了 3 周——是项目最慢的部分</u>，也是最被玩家记住的部分。

### 5.3 战斗节奏

每场战斗 3-6 回合。每回合 1 个决策。**总时长 2-3 分钟**——这是关键，太长玩家会疲劳。

## 六、0 点碎片与收藏册

### 6.1 0 点碎片（共 7 块）

散落在 7 个特殊地点。每块碎片解锁一个**永久技能**和一段**世界观故事**：

| 碎片 | 技能 | 故事主题 |
|---|---|---|
| 第 1 块 | 油量 +20% | 核战前的加油站 |
| 第 2 块 | 战斗前先手 | 军事基地的广播 |
| 第 3 块 | 乘员血量 +1 | 医院的最后一刻 |
| 第 4 块 | 派系任务 +1 派系声望 | 商会的暗号 |
| 第 5 块 | 改装槽位 +1 | 工厂车间的蓝图 |
| 第 6 块 | 隐藏商人提示 | 走私者的口信 |
| 第 7 块 | 真结局解锁 | 0 点的真相 |

> 7 块不是随机数。<u>为什么是 7</u>：核战前的世界据说有 7 个"0 点"——核弹落地的 7 个瞬间。碎片数量本身是叙事的一部分。

### 6.2 收藏册

- 50+ 货物图鉴（每种货物有来源、价格、用途）
- 30+ 城镇图鉴（每个城镇有 4-5 个标志性场景）
- 20+ 事件图鉴（玩家见过的事件会记录在册）

> 收藏册是"<u>奖励探索，不奖励战斗</u>"。<u>很多玩家是冲着 100% 完成度玩第二轮第三轮的</u>，这比"再来一局挑战"更长期。

## 七、城镇与地图

### 7.1 地图生成

12-18 个城镇随机生成在世界地图上。地形分 3 类：
- **公路**：油耗低、事件少
- **荒漠**：油耗高、有辐射区
- **山区**：慢但安全

**踩过的坑**：早期地图是"12 个固定点"，玩家第二周就背下来了。后来改成"种子生成"，每次新游戏地形不同。<u>但城镇名字和派系位置固定</u>——因为世界观是"同一个世界不同条线"，不是"完全随机"。

### 7.2 城镇设计

每个城镇 4-5 个标志性地点：
- 商店（油/弹药/零件/货物）
- 旅店（招募临时乘员 + 存档）
- 派系据点（接主线任务）
- 0 点碎片（部分城镇）
- 特殊事件（每个城镇 1 个独特故事）

> 早期每个城镇都一样（商店 + 旅店 + 派系据点）。后来加"<u>每个城镇 1 个独特故事</u>"——比如"永夜城"（永远停电，只有晚上能进的赌博店）。<u>这种独特性让玩家想"逛遍所有城镇"</u>。

## 八、UI 关键决定

- **像素字体**（5x7 bitmap 自绘）：不用 Web 字体，所有文字像素渲染（参考《极乐迪斯科》）
- **640x480 画布**：复古分辨率，向上扩展
- **4 个主面板**：菜单 / 城镇 / 地图 / 战斗
- **无教程**：第一回合选任务时提示一次"<u>不同派系声望影响任务报酬</u>"，其他全部靠玩家摸索
- **即时存档**：每回合结束自动存。死亡不丢档（这是 roguelike 玩家的痛点）

> UI 原则："<u>每一个按钮都在回答玩家的下一个问题</u>"。玩家不会主动看 UI 文档，他们只看"屏幕上现在发生了什么"。

## 九、商业化

**纯买断制**，定价：
- **Steam 完整版 38 元**（含全部 4 派系 + 0 点碎片 + 收藏册）
- **GOG 38 元**（无 DRM 版本，给不喜欢 Steam 的玩家）
- **Itch.io 自由定价**（含 5 美元建议价）

**不做内购**：roguelike 玩家对"内购影响公平"零容忍（参考《Hades》社区对任何"氪金复活"的反应）。

**DLC 计划**：
- DLC 1：**新派系"零派"**（中立神秘组织，2026 Q4）
- DLC 2：**困难模式"铁锈之路"**（车辆会持续损耗，2027 Q1）
- DLC 3：**创意工坊**（玩家自制载具/任务，2027 Q3）

> 创意工坊是"<u>延长游戏寿命的最高性价比方式</u>"——但需要先在 Steam 拿到 5 万+ 评测才会做。

## 十一、踩过的坑

1. **派系声望是"线性增长"** → 改为"双向 ±值"。早期声望 0-100，砍一刀变 -100 到 +100，**这让玩家能"做错事"产生真正的紧张感**。
2. **油量上限 100、起步 50** → 起步改 100。<u>为什么</u>：起步 50 让玩家第一回合就在想"会不会油不够"，但 roguelike 第一回合应该是"上手"而不是"紧张"。
3. **战斗时长 5-8 分钟** → 砍到 2-3 分钟。<u>踩的坑</u>：第一版战斗太复杂，8 个回合 + 技能树。玩家说"我玩 3 局就累了"。**精简到 3 战术 + 3 回合，反而策略性更强**（决策少 = 每个决策重要）。
4. **城镇 12 个完全相同** → 每个加 1 个独特故事。<u>这一步</u>：让"逛遍地图"成为可玩目标。
5. **没有"派系间冲突"** → 加了任务里的派系间对立。<u>早期任务都是"单派系"</u>，改成"钢铁教会 vs 自由民"的二选一任务后，玩家选择率从 30% 涨到 80%。
6. **教程 3 屏** → 砍到 0 屏，只在第一回合弹 1 句提示。<u>roguelike 玩家讨厌被"教"</u>，他们喜欢自己琢磨。

> 教训：**<u>派系系统的核心是"双向选择"，不是"单边任务"</u>**。当一个任务让玩家必须选边时，玩家的代入感提升 3 倍（基于测试组的反馈）。

## 十二、待解决问题（开放讨论）

1. **真结局内容是什么？** 7 块碎片都收集后的剧情还没写。**目前占位是"揭示 0 点真相"**，但具体是什么还没想好。
2. **是否要做 mod 工具？** 创意工坊需要 mod 支持，但 mod 工具要单独做（2-3 周）。<u>倾向"先做关卡 JSON 导出，玩家自改"</u>。
3. **第 5 个派系？** 4 派系平衡但**<u>世界观的"神秘侧"</u>** 只有 0 点碎片，缺一个对立的派系。**可能 DLC 加"零派"**。
4. **是否要做联机？** Roguelike 联机要么做"好友互送货物"，要么做"组队跑货"。**风险高、收益不一定高**，目前**搁置**。
5. **海外发行策略？** 国内 Steam + GOG + Itch。海外**只上 Steam + GOG**（海外玩家用 Itch 少）。**翻译**：英文必做、日文考虑中（像素 roguelike 日本市场大）。
