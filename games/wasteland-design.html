这个游戏想解决的是：Roguelike怎么做出"规划感"？
不是走一步看一步，是走之前就已经想好了这条路线值不值得走。

🎮 先去玩一下

这个游戏要解决什么问题

玩过很多废土题材的Roguelike，基本上都是一个模式：遇到啥就打啥，打赢了捡东西，打输了重来。没有"规划"这回事，运气成分太大。

我想做的不是随机遭遇，而是一个有"路线规划"感的游戏——玩家在出发前就能看到这条路线值不值得走、有什么风险、预期收益是多少。运气依然存在，但玩家能用规划来对抗运气，而不是完全被运气支配。

设计目标

  • 路线信息完整透明——玩家能看到路线危险度、预期耗时、沿途资源点，而不是开盲盒
  • 改装系统提供真实策略——不是越多越好，而是"这条路我需要防御，那个任务我需要速度"
  • 经济系统有套利空间——不同城镇物价不同，低买高卖是核心盈利手段，但存在市场波动

核心系统是怎么设计的

1. 程序化地图生成

地图不是随机撒点，是用最小生成树算法生成的——确保任意两个城镇之间都有可达路径，同时地图上存在"快捷路线"的探索空间。玩家可以选择走主路（安全但收益低）或探索捷径（危险但可能有意外收获）。

城镇分5类：安全区（无战斗）、资源镇（低价出售特定商品）、黑市（可买到其他地方买不到的东西）、危险镇（怪物密集但奖励高）、废墟（完全随机，高风险高回报）。每次重开城镇类型和位置都会变化，但类型分布比例保持稳定。

2. Deck Building 改装系统

改装不是"买装备"，而是"构建你的卡组"。货车有5个槽位，每个槽位对应不同功能区域（动力系统、防护系统、武器系统、储物系统、通讯系统）。每个槽位可以装多个零件，零件以"卡"的形式在旅途中打出——某些情况自动触发。

// 零件配置表示例

{
  "id": "part_heavy_armor",
  "name": "重型护甲板",
  "slot": "defense",
  "effect": { "armor": +30, "speed": -15 },
  "rarity": "rare",
  "price_range": [80, 120]
}

5套装的逻辑是：同套装零件达到一定数量会激活套装效果。比如"废土求生套装"凑齐3件后，燃油消耗降低20%——鼓励玩家不要只追求单件数值，而是考虑整体构建思路。

3. 经济系统

10种商品在不同城镇有不同价格，且价格会随玩家行为波动——如果玩家大量购买某商品，该商品在这个城镇的价格会上涨。设计目的是让玩家需要"记住"自己在哪些城镇买了什么，而不是无脑全买。

三重消耗（燃油、耐久、理智）形成决策约束：跑更多路线意味着更多消耗，但不跑就没钱赚。玩家需要在"跑更多"和"控成本"之间找到平衡点。

商品套利示例

安全区A镇：绷带 5金币 ← 便宜产地
资源镇B镇：绷带 8金币
黑市C镇：绷带 15金币 ← 高需求地
玩家在A镇买入10个绷带（50金），去C镇卖出（150金），一趟净赚100金

4. 旅途事件系统

旅途事件分三类：遭遇战（触发战斗）、随机事件（商人、流浪者、天气变化等）、资源点（自动发现额外物资）。三类事件根据路线危险度动态配比——危险度越高，遭遇战概率越大，但也更容易触发稀有事件。

系统之间的关系

地图生成系统

决定城镇分布、路线结构、危险度分布，为玩家提供"规划原材料"

经济系统

管理物价、交易逻辑、三重消耗，是玩家"规划"的核心约束

改装系统

决定玩家的能力边界，坦克型玩家适合高危路线，商人型玩家适合低危跑商

旅途事件系统

读取地图系统的路线信息，在旅途中插入随机变量，制造"规划之外"的变化

核心循环：玩家查看地图（地图系统）→ 规划路线（结合经济系统）→ 改装车辆（改装系统）→ 出发 → 旅途事件随机介入 → 回到城镇交易 → 更新改装 → 再出发。

后续想做的

v1.1 · 货运任务系统

从"自由跑商"升级到"任务+自由混合"。NPC发布货运任务（指定起点终点、有时间限制、奖励加成），玩家可以选择接任务（有目标感）或继续自由跑商（有灵活性）。任务失败有惩罚，但奖励也更丰厚。

v1.2 · 城镇关系系统

城镇之间有派系关系（比如A镇和B镇是敌对关系，玩家在A镇买的东西运到B镇卖会触发特殊事件）。派系关系会随游戏进程变化，玩家可以主动介入派系冲突，用自己的货运能力改变废土的势力格局。

DLC · 多人联机货运

一个玩家负责开车护送，另一个玩家扮演"废土电台"提供情报——哪里有危险、哪个城镇缺货、哪个路线被封锁。联机不改变核心玩法，而是增加了一个"协作规划"的维度。
