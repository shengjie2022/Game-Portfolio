一、时间交易循环

游戏核心循环设计：每日接待客人 → 进入记忆选择 → 产生涟漪 → 推进剧情。通过时间约束制造紧迫感，通过蝴蝶效应制造因果感。

-8碎片

设计意图

时间碎片每日消耗 8 点的设计，让 30 天总时长形成天然压力。访客队列让玩家需要在"今天见谁"上做策略选择，而不是无脑全接。老客户优先机制增强叙事连贯性。

二、每日访客队列系统

队列系统让玩家在"接待谁"上产生策略决策，超出上限的客人进入明日队列，老客户拥有优先权。

// 队列管理数据结构 tomorrowQueue: string[] // 明日待接待客人ID列表 dailyCount: number // 今日已接待人数 // 核心函数 addToQueue(clientId) // 添加到明日队列 processDailyQueue() // 每日开始时读取队列 sortByPriority() // 老客户优先排序

队列规则

  • 每日最大接待量：3人
  • 超限自动进入明日队列
  • 老客户（下周回归）优先于新客户
  • 新周目队列重置

设计意图

  • 制造"今天见谁"的策略决策
  • 队列预览仅显示图标，保持悬念
  • 老客户优先增强叙事记忆感
  • 为隐藏角色设置前置条件

三、时间紧迫度系统

三级紧迫度状态，通过视觉+BGM双重反馈，让玩家直观感知时间压力。

剩余天数 紧迫度 视觉效果 音效
> 10天 正常 无特殊效果 常规BGM
6–10天 注意 橙色边框 常规BGM
≤5天 紧迫 红色边框 + 脉冲动画 + 图标闪烁 紧迫版BGM
// 当 timePressure 进入"紧迫"状态 if (remainingDays <= 5 && currentBGM !== 'urgent') { fadeOutCurrentBGM(1000); // 渐出当前BGM playBGM('urgent_shop'); // 切换紧迫版BGM }

设计洞察

紧迫度切换使用 1 秒渐变而非突然切换，BGM 平滑过渡让玩家感知变化而非被惊吓。三级状态（正常→注意→紧迫）让玩家有心理预期，不会产生"突然死亡"的挫败感。

五、跨周目MetaSave系统

双层存档架构：局内存档（单周目）+ MetaSave（跨周目持久化）。新周目读取 MetaSave，解锁隐藏内容。

// MetaSave 数据结构 MetaSave { // 基础字段 currentDay: number, timeFragments: number, ripples: string[], completedCharacters: string[], // v4.3 新增字段 chenboReunionSeen: boolean, // 陈波重逢结局 chenboAfterSeen: boolean, // 陈波 After 结局 shiyurenUnlocked: boolean, // 失忆者解锁 shiyurenEndingsSeen: object, // 失忆者结局记录 dailyQueueSeen: array, // 今日已见队列 shiyurenTrackCompleted: boolean // 失忆者故事线完成 }

局内存档

单周目内的游戏进度：当前天数、时间碎片、已接待角色。存储在 localStorage，随周目重置。

MetaSave

跨周目持久化数据：涟漪记录、结局解锁、隐藏角色状态。新周目读取此数据，实现"记忆延续"。
